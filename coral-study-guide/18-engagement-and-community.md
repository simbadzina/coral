# 18 — Engagement and community

How to be visible, how to ask, how to keep up with Coral.

## Maintainers

Get the list from `git shortlog -sn --since="2 years ago"` for the current active set. Coral is mostly maintained by LinkedIn engineers; reviews tend to be quick during US business hours.

## Slack

The public Coral Slack workspace invite is in the repo `README.md` (Slack section). Use it for design questions before opening a PR; maintainers answer in-thread.

## CONTRIBUTING.md

- Read `CONTRIBUTING.md` at the repo root before your first PR.
- The CLA is in the same file — sign before submission.

## Release cadence

- Shipkit auto-generates release notes from PR titles. Write good PR titles.
- Releases land roughly weekly when there are merges. Check the `Releases` page on GitHub.

## Issue tracker

GitHub Issues. Many are tagged by module. Issues #575 (CoralCatalog migration), #559 and similar are good places to see active design conversations.
